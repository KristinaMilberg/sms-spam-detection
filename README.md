# SMS Spam Detection

NLP-модель для классификации мошеннических SMS-сообщений на основе TF-IDF и Logistic Regression.

## Задача

Бинарная классификация текстов: отличить обычные сообщения (ham) от спама/фишинга.  
Датасет: [SMS Spam Collection](https://www.kaggle.com/datasets/uciml/sms-spam-collection-dataset) — 5 572 сообщения, 13% спам.

## Стек

`Python` `Pandas` `scikit-learn` `NLTK` `Matplotlib`

## Результаты

| Модель | Precision (spam) | Recall (spam) | F1 (spam) |
|--------|-----------------|---------------|-----------|
| Naive Bayes (baseline) | 1.00 | 0.78 | 0.88 |
| Logistic Regression (baseline) | 1.00 | 0.78 | 0.88 |
| **LR + длина + порог 0.3** | **0.96** | **0.89** | **0.92** |

## Ключевые находки

- Спам-сообщения значительно длиннее обычных: медиана **149 vs 52 символа**
- Топ-маркеры спама: `free`, `£`, `call`, `txt` — типичный паттерн фишинга
- Добавление признака длины и снижение порога с 0.5 → 0.3 подняло Recall с 0.78 до **0.89**
- Настройка порога позволяет гибко управлять балансом Precision/Recall под бизнес-задачу

## Связь с антифродом

Подход переносится на детекцию фишинговых коммуникаций: TF-IDF + n-граммы эффективно работают на коротких текстах, а настройка порога классификации позволяет управлять чувствительностью системы в зависимости от бизнес-требований.
